Panduan Pembuatan Nova Robot Assistant

Dokumen ini ditujukan bagi pengembang maupun pemula yang ingin membangun robot asisten pintar berbasis kecerdasan buatan dari awal. Panduan mencakup penyiapan lingkungan sistem, instalasi dependensi, hingga menjalankan aplikasi di komputer lokal atau Raspberry Pi.

1. Penyiapan Python di Sistem Operasi

Aplikasi ini membutuhkan Python versi 3.10 atau versi di atasnya. Cari bagian di bawah sesuai dengan sistem operasi yang Anda pakai.

Windows

  1. Unduh installer Python resmi dari python.org. Disarankan memilih versi 3.10 atau 3.11.
  2. Buka file installer yang sudah diunduh. Centang kotak "Add Python to PATH" sebelum mengeklik tombol "Install Now".
  3. Untuk memastikan instalasi berhasil, buka Command Prompt (CMD) dan jalankan: 
    python --version

macOS

  1. Instal Homebrew jika belum terinstal pada Mac Anda dengan perintah terminal: 
    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
  2. Pasang Python menggunakan perintah brew: 
    brew install python@3.10
  3. Uji versi terinstal di Terminal: 
    python3 --version

Linux dan Raspberry Pi

  1. Buka jendela Terminal dan perbarui indeks paket sistem Anda: 
    sudo apt update && sudo apt upgrade -y
  2. Instal Python, pengelola paket PIP, serta modul lingkungan virtual dengan perintah berikut: 
    sudo apt install python3 python3-pip python3-venv -y

2. Instalasi Dependensi Pustaka Sistem

Karena aplikasi memproses input suara dari mikrofon (STT) serta mengeluarkan ucapan suara (TTS) dan memutar berkas musik secara lokal, Anda perlu memasang beberapa alat multimedia bawaan sistem operasi.

Windows

  • FFmpeg: Unduh paket biner dari situs resmi ffmpeg.org, ekstrak folder hasil unduhan, dan masukkan folder bin ke dalam Environment Variables sistem agar bisa dibaca Python.
  • Build Tools C++: Jika terjadi kendala saat mengompilasi modul luring (seperti PyAudio), unduh dan instal Build Tools untuk Visual Studio dari situs web Microsoft.

macOS

Jalankan perintah Homebrew untuk memasang komponen audio eksternal:

brew install portaudio ffmpeg mpg123

Linux dan Raspberry Pi

Pasang semua pustaka audio dan pemutar berkas mp3 di terminal:

sudo apt install portaudio19-dev ffmpeg mpg123 libasound2-dev -y
Catatan Penggunaan Raspberry Pi
Jika Anda memasang proyek ini di Raspberry Pi, Anda disarankan menggunakan mikrofon USB portabel dan menyetel output audio default pada lubang Jack 3.5mm atau konektor HDMI audio yang aktif.

3. Daftar Struktur Berkas Proyek

Berikut adalah peta direktori proyek asisten suara Nova yang perlu Anda siapkan:

asisten-nova/ ├── main.py # Server FastAPI untuk websocket & routing perintah ├── gemini_bot.py # Logika kecerdasan buatan Gemini AI & API Call ├── stt.py # Konversi suara mikrofon menjadi teks (luring via Whisper) ├── tts.py # Sintesis ucapan teks menjadi suara lisan (luring via pyttsx3) ├── database.py # Pengelola database SQLite lokal (catatan, alarm, tugas) ├── music_player.py # Pemutar file audio lokal menggunakan utilitas mpg123 ├── desktop_app.py # Pembungkus antarmuka web ke aplikasi window (pywebview) ├── requirements.txt # Daftar modul Python eksternal ├── templates/ │ └── index.html # Tampilan antarmuka grafis robot dan chatbot └── static/ ├── app.js # Logika frontend, websocket, dan clock timer └── style.css # Pengaturan tata letak dan desain visual dashboard

4. Pembuatan Virtual Environment dan Instalasi Pustaka

Lingkungan virtual berguna agar modul eksternal tidak bentrok dengan modul Python global di komputer Anda.

  1. Buka terminal atau CMD pada direktori proyek Anda.
  2. Buat lingkungan virtual dengan perintah berikut: 
    python3 -m venv venv
  3. Aktifkan lingkungan virtual:

    Windows (Command Prompt):

    venv\Scripts\activate

    macOS / Linux / Raspberry Pi:

    source venv/bin/activate
  4. Instal modul-modul yang dibutuhkan melalui berkas requirements.txt: 
    pip install --upgrade pip
pip install -r requirements.txt
Mengatasi Kegagalan Instalasi PyAudio di Windows
Jika Anda mendapati error "pyaudio installation failed", gunakan paket pipwin untuk membantu pemasangan otomatis file biner PyAudio yang sesuai: 
pip install pipwin
pipwin install pyaudio

5. Konfigurasi Kunci API Gemini

Untuk menghubungkan Nova dengan AI Gemini di cloud, Anda memerlukan kunci otorisasi API resmi.

  1. Kunjungi portal pengembang Google AI Studio.
  2. Buat kunci baru melalui menu "Get API Key".
  3. Setelah menyalin kunci tersebut, pasang sebagai variabel sistem di komputer Anda:

Untuk Windows (Command Prompt):

setx GEMINI_API_KEY "KUNCI_API_ANDA_DISINI"

Untuk macOS / Linux (Terminal):

export GEMINI_API_KEY="KUNCI_API_ANDA_DISINI"

Simpan baris perintah ekspor tersebut di akhir file konfigurasi terminal seperti ~/.bashrc atau ~/.zshrc Anda agar kunci tidak terhapus ketika jendela terminal ditutup.

6. Cara Menjalankan Aplikasi

Anda dapat mengoperasikan Nova Robot Assistant dalam dua bentuk:

A. Mode Server Web

Jalankan server lokal FastAPI di dalam terminal:

python3 -m uvicorn main:app --host 127.0.0.1 --port 8005

Setelah server aktif, buka web browser Anda dan akses alamat berikut:

http://127.0.0.1:8005

B. Mode Jendela Aplikasi Desktop

Jika ingin menjalankan Nova dalam bentuk jendela aplikasi mandiri, jalankan skrip desktop pembungkus berikut:

python3 desktop_app.py
Catatan Pengunduhan Model Whisper Pertama Kali
Saat pertama kali aplikasi mendeteksi perintah suara, sistem akan mengunduh model pemrosesan ucapan Whisper (model berukuran kurang lebih 500 MB) secara otomatis. Proses ini membutuhkan jaringan internet yang stabil di awal peluncuran.

7. Solusi untuk Masalah yang Sering Terjadi

  • Masalah: "WebSocket Terputus" berulang kali di antarmuka
    Solusi: Periksa konsol log terminal tempat Anda menjalankan FastAPI. Pastikan Anda telah menetapkan variabel lingkungan GEMINI_API_KEY dengan benar dan model Whisper telah selesai terunduh.
  • Masalah: Mikrofon tidak menangkap suara
    Solusi: Pastikan mikrofon fisik Anda telah dicolokkan ke komputer dan diatur sebagai perangkat input audio default pada pengaturan sistem suara Anda. Izinkan akses mikrofon bila diminta oleh peramban.
  • Masalah: Tidak terdengar suara balasan dari asisten
    Solusi: Modul TTS lokal menggunakan mesin suara bawaan sistem. Di Linux, pastikan program espeak terinstal (sudo apt install espeak).
